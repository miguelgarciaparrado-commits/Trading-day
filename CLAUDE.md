## CRITICAL: Build Pipeline

**Vercel/Netlify serves static HTML — there is NO build step on the hosting platform.**

Every code change requires:
```bash
node build.js               # JSX → trading-diary.html
cp trading-diary.html index.html  # index.html is what the root URL serves
git add trading-diary.jsx trading-diary.html index.html
git commit -m "..."
git push -u origin <branch>
```

**Never** push only the `.jsx` file — the deployed app will not change.

### How `build.js` works

1. Reads `trading-diary.jsx`, strips `import`/`export` statements
2. Compiles with TypeScript API: `jsx: React, target: ES2020, module: None`
3. Post-processes: removes `"use strict"`, `exports.*`, `Object.defineProperty(exports...)` artifacts
4. Removes orphan commas (TypeScript bug: treats `<` `>` in JSX as generics, inserts trailing commas)
5. Prepends `const {useState,useEffect,useRef,useCallback} = React;`
6. Splices compiled JS into `trading-diary.html` between two fixed markers:
   - Start: `const {useState,useEffect,useRef,useCallback} = React;`
   - End: `var root = ReactDOM.createRoot(document.getElementById("root"));`

`node --check trading-diary.html` does **not** work (Node.js rejects `.html` extension). To verify JS syntax, extract the script block manually.

### JSX Constraints (TypeScript/UMD React, no Babel)

- No `?.` optional chaining — use `(x && x.prop) || default`
- No `??` nullish coalescing — use `|| default`
- No `async ({destructured}) =>` in parameters — use `async function(param){ const {a,b}=param; }`
- No IIFEs `(()=>{...})()` inside JSX — extract to named function
- No template literal types or TypeScript generics
- All JSX must have balanced divs per conditional branch

## Architecture

### Stack
- **React 18 UMD** (loaded from CDN, no bundler)
- **CSS-in-JS** — all styles are inline objects in `const S = {...}` at module level
- **Binance REST + WebSocket** — price data and real-time klines
- **Supabase** — primary persistence (PostgreSQL, single row per user)
- **localStorage** — offline backup, always written synchronously before Supabase
- **Anthropic API** (Claude) — chat tab + profile analysis generation

```
Edit → localStorage (immediate) → Supabase (600ms debounce)
Load → Supabase (priority) → localStorage (fallback)
```

## Position System

### Position Object Schema
```javascript
{
  id: number,
  asset: string,          // e.g. "BTC", "SOL"
  dir: "Long" | "Short",
  capital: number,        // USD capital
  entry: number,          // entry price
  sl: number,             // stop loss price
  tp: number,             // single take profit (OR use tpLevels)
  tpLevels: [             // partial takes (ventas parciales)
    { id, price, pct, hit }  // pct = % of position to close at this level
  ],
  capitalRemaining: number, // after partial takes
  be: boolean,            // breakeven flag
  date: string,
  note: string
}
```

### Direction Validation Rules
- **LONG**: SL < entry, TP > entry
- **SHORT**: SL > entry, TP < entry
- Always use `Math.abs()` for dollar amounts; validate sides separately

### Auto-Close Logic (`autoCheckPositions`)
Runs after every `fetchPrices()` cycle (~60s). Checks all open positions:
- SL hit → closes position, saves to `xhist` with "🛑 SL auto" note
- TP level hit → closes that % of position, marks level `hit:true`
- All TP levels hit → removes position entirely
- Sends Telegram + browser notification via `notifyAutoClose()`

### Ratio R:R Tracking
When a position closes (any method), `ratio` is saved in the `xhist` entry and accumulated in `ps.ratioSum` / `ps.ratioCount`. The Resumen tab shows "RATIO R:R MEDIO".

## Alert System (AlertasTab)

### How Alerts Work
Each alert = one Binance WebSocket kline stream (`symbol@kline_interval`).
Historical candles pre-loaded via REST on start (250 candles).
OHLC stored in `ohlcRef.current[symbol+interval]`.
RSI history stored in `rsiHistRef.current[symbol+interval]`.
EMA state stored in `emaStateRef.current`.

### Automatic Triggers (no user toggle required)
- RSI enters ≤30 zone → `"rsi_oversold"` (fires once on zone entry, not per-candle)
- RSI enters ≥70 zone → `"rsi_overbought"`
- EMA 7 crosses above EMA 25 → `"golden"`
- EMA 7 crosses below EMA 25 → `"death"`
- EMA 50 crosses above EMA 200 → `"ema200_golden"`
- EMA 50 crosses below EMA 200 → `"ema200_death"`
- RSI divergence on closed candle → `"rsi_div_bull"` or `"rsi_div_bear"`

### Optional User-Configured Triggers
- Custom RSI level (`rsiCustomEnabled`, `rsiCustomTarget`, `rsiCustomCondition`)
- Channel boundary touch (`channelEnabled`)
- FVG covered (`fvgEnabled`)

### Cooldown / Anti-spam
`lastTrigRef.current[key]` tracks fired state per alert+type. Fires once when condition becomes true, resets when it becomes false. Zone-based cooldown for RSI 30/70 (tracks `"oversold"/"overbought"/"neutral"` state).

### Alert Object Schema
```javascript
{
  id: number,
  symbol: string,           // e.g. "BTCUSDT"
  label: string,            // display name
  interval: "1h"|"4h"|"1d"|"1w",
  rsiCustomEnabled: boolean,
  rsiCustomTarget: number,  // 31-69
  rsiCustomCondition: "above"|"below",
  channelEnabled: boolean,
  fvgEnabled: boolean,
  active: boolean,
  currentRsi: number|null,
  currentPrice: number|null,
  error: boolean
}
```

### Notifications
Every `sendAlert()` call:
1. Logs to in-app `logs` state (shown in Alertas tab)
2. Sets `lastAlert` state (banner shown for 8s)
3. Vibrates device (if supported)
4. Sends Telegram message (if token + chat ID configured)
5. Shows browser Notification (if permission granted)

## Profile Score System

`calcScore(ps, pats, jnl)` returns 0-100:
- **s1** (30 pts): SL discipline — `slOk / (slOk + slBroken)`
- **s2** (20 pts): TP discipline — reaching objective vs early close
- **s3** (20 pts): No revenge trading — `20 - revenge * 5`
- **s4** (20 pts): Pattern book usage — confirmed patterns × 7
- **s5** (10 pts): Diary activity — wins + lessons + mistakes
- **bon** (5 pts): Journal entries linked to closed positions

Level labels: Aprendiz (<40) → En Desarrollo (40-59) → Consistente (60-74) → Avanzado (75-89) → Profesional (≥90)


### Branch Strategy
- Feature branches: `claude/fix-*` or `claude/feat-*`
- Merge to `main` via PR (GitHub MCP tools available)
- **Always build and commit both HTML files before pushing**

### Debugging Tips
- If deployed app shows old version: hard refresh (Ctrl+Shift+R), check that `index.html` was rebuilt
- If data appears lost: check browser localStorage (`td-user` key), may be browser cache issue
- Runtime errors won't show in `node --check` — test in browser console
- The `save()` function debounces 600ms to Supabase; `SPos()` writes localStorage synchronously first
